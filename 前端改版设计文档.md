## 1. 项目背景与目标

当前前端：两个内联 HTML 页面（`AGENT_GUIDE_HTML` 和 `CONFIG_HTML`）嵌入在 `asr_api_server.py` 中，根路由 `/` 302 重定向到 `/guide`。

改版目标：
- 主页变成**文件上传入口**，支持拖拽 + 自动上传
- 上传后**实时展示转写内容**（轮询方式，可选功能）
- 支持**选择输出文档保存目录**
- 支持**预选多个视频**的上传队列（排队依次处理）
- 所有已有功能（健康检查、ASS 下载、LLM 润色、Config 配置、Guide 指南）通过页面内按钮/导航访问
- 纯前端 SPA，无框架，无构建步骤，暗色主题，中文界面

参考项目：`D:\work\main (1)\transby2-main` — 其队列式线程安全 UI 更新模式、预设管理系统、以及自适应合并逻辑为本设计提供了参考。

## 2. 总体架构

### 2.1 前端方案

**SPA（Single Page Application）**，使用 hash-based 路由，纯 vanilla HTML/CSS/JS（ES6 modules）。由 FastAPI 以静态文件方式挂载。

- 路由方案：`window.location.hash` + `hashchange` 事件
- 模块系统：ES6 `type="module"` 的 `<script>` 标签
- 数据持久化：`localStorage`
- 无外部依赖、无 npm、无 build 步骤

### 2.2 FastAPI 静态文件服务

```python
# 在 asr_api_server.py 末尾、if __name__ 之前添加

from fastapi.staticfiles import StaticFiles

FRONTEND_DIR = BASE_DIR / "frontend"

# 1. 挂载静态资源（CSS/JS）
app.mount("/assets", StaticFiles(directory=str(FRONTEND_DIR)), name="assets")

# 2. 将 /guide 和 /config 重定向到 SPA
@app.get("/guide")
async def guide_redirect():
    return RedirectResponse(url="/#guide")

@app.get("/config")
async def config_redirect():
    return RedirectResponse(url="/#config")

# 3. 根路由直接返回 index.html（不再重定向）
@app.get("/")
async def serve_spa():
    return HTMLResponse((FRONTEND_DIR / "index.html").read_text(encoding="utf-8"))

# 4. 移除内联的 AGENT_GUIDE_HTML 和 CONFIG_HTML 字符串（约 400 行）
```

### 2.3 SPA 路由表

| Hash | 页面 | 模块 |
|------|------|------|
| `#/upload` | 上传队列 + 实时转写（默认页） | `pages/upload.js` |
| `#/results` | 转写结果历史 | `pages/results.js` |
| `#/config` | LLM 配置 | `pages/config.js` |
| `#/ass` | ASS 字幕生成 | `pages/ass.js` |
| `#/refine` | LLM 润色 | `pages/refine.js` |
| `#/health` | 服务器状态 | `pages/health.js` |
| `#/guide` | AI Agent 指南 | `pages/guide.js` |

## 4. 后端改造点

### 4.1 进度持久化（关键改动）

当前：转写完成后进度条目被**删除**，前端无法轮询到已完成的结果。

改造：
- 完成后条目保留在 `_transcription_progress` 中，`status="completed"`
- 添加 `_updated_at` 时间戳字段
- 添加后台清理任务：每 60 秒扫描一次，删除超过 10 分钟的 completed/error 条目
- 完成时存储 `full_text`、`segments`、`filename` 到进度字典

```python
# _update_progress 中增加 _updated_at
def _update_progress(file_id: str, **kwargs):
    with _progress_lock:
        if file_id not in _transcription_progress:
            _transcription_progress[file_id] = {"words": [], "chunks_done": 0, "total_duration": 0.0}
        _transcription_progress[file_id].update(kwargs)
        _transcription_progress[file_id]["_updated_at"] = time.time()

# _transcribe_file 中，完成时存储完整数据
_update_progress(file_id, status="completed", full_text=full_text, segments=segments,
                 words=all_words, duration_sec=round(total_duration, 2), filename=filename)
# 不再 pop 删除！
```

### 4.2 排队系统

GPU 一次只能处理一个转写任务，需排队。

```python
# 新增全局变量
_processing_queue: list = []   # [{file_id, input_path, ext, language, context, max_chars, pause_threshold}]
_queue_lock = threading.Lock()
_is_processing = False

def _queue_next():
    """后台处理队列中的文件，一次一个"""
    global _is_processing
    while True:
        with _queue_lock:
            if not _processing_queue:
                _is_processing = False
                return
            item = _processing_queue.pop(0)
        _is_processing = True
        _update_progress(item["file_id"], status="transcribing")
        # ... 调用 _transcribe_file ...
        _is_processing = False
```

### 4.3 新增端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/transcribe_async` | POST | 非阻塞上传 + 排队，立刻返回 `{"file_id", "position", "status": "queued"}` |
| `/output_dir` | POST | 设置输出目录 |
| `/output_dir` | GET | 获取当前输出目录 |
| `/save_result` | POST | 将转写结果保存到输出目录 |

### 4.4 保留旧端点

`/transcribe`（阻塞）、`/transcribe_ass`、`/refine`、`/full_pipeline` 等原有端点全部保留，AI agent 和 curl 用户继续使用。

## 6. 数据流与轮询策略

### 6.1 转写生命周期

```
用户拖入文件 → 添加到队列（状态: 等待中）
用户点击"开始处理"
  → POST /transcribe_async → 返回 {file_id, position, status: "queued"}
  → 服务器后台排队，轮到后开始处理（状态: 转写中）
  → 前端开始轮询 /transcribe_status/{file_id}（每 1 秒）
  → 每个轮询返回 {words, chunks_done, total_duration, status}
  → 前端将新词追加到实时展示
  → status = "completed" → 停止轮询 → 显示完整文本
  → 保存到 localStorage → 队列中下一个文件自动开始
```

### 6.2 轮询策略

| 场景 | 行为 |
|------|------|
| 活跃转写中 | 每 1 秒轮询 `/transcribe_status/{file_id}` |
| 队列中有等待项 | 每 5 秒轮询 `/transcribe_list` 查看队列状态 |
| 标签页被隐藏 | 降频到每 10 秒一次（`document.visibilityState`） |
| 网络请求失败 | 重试 3 次（间隔 1s, 2s, 4s），3 次全失败后显示错误 |
| 用户导航离开 | `onLeave()` 停止所有轮询，返回时 `onEnter()` 重新检查 |
| 页面刷新 | 加载时检查 `/transcribe_list` 恢复状态 |

### 6.3 localStorage 数据模型

```javascript
// Key: 'asr_llm_config'（向后兼容旧版本）
{ api_url, api_key, model_name, system_prompt, temperature: 0.3 }

// Key: 'asr_results'
[{ id, filename, date, duration_sec, language, text, segments, words }]

// Key: 'asr_preferences'
{ output_dir, default_language, default_max_chars: 50, default_pause_threshold: 0.3 }
```

## 8. API 集成参考表

### 各组件使用的端点

| 组件 | 端点 | 方法 | 轮询频率 |
|------|------|------|----------|
| 上传队列 | `/transcribe_async` | POST | 无 |
| 上传队列 | `/transcribe_list` | GET | 有活跃任务时每 5 秒 |
| 实时展示 | `/transcribe_status/{file_id}` | GET | 转写中每 1 秒 |
| 结果 | `/save_result` | POST | 无 |
| 结果 | `/output_dir` (GET) | GET | 加载时 |
| 配置 | `/llm_test` | POST | 无 |
| ASS | `/transcribe_ass` | POST | 无 |
| 润色 | `/refine` | POST | 无 |
| 健康 | `/health` | GET | 每 30 秒 |
| 导航状态灯 | `/health` | GET | 每 30 秒 |
| 指南 | `/health`（获取 IP） | GET | 加载时 |

### 状态流转

```
queued → transcribing → merging → completed → (10 分钟后自动清理)
  ↓          ↓                      ↓
error      error                  error
```

### 新端点请求/响应格式

```
POST /transcribe_async
  Request: multipart/form-data { file, language, context, max_chars, pause_threshold }
  Response 200: { "file_id": "a1b2c3d4", "position": 1, "status": "queued" }

GET /transcribe_status/{file_id}
  Response 200 (转写中): { "status": "transcribing", "words": [...], "chunks_done": 5, "total_duration": 180.5 }
  Response 200 (已完成): { "status": "completed", "full_text": "...", "segments": [...], "filename": "lecture.mp4" }
  Response 200 (已清理): { "status": "not_found" }

POST /output_dir
  Request: application/x-www-form-urlencoded { path: "D:/transcriptions" }
  Response 200: { "success": true, "path": "D:/transcriptions" }

POST /save_result
  Request: application/x-www-form-urlencoded { file_id, filename: "lecture.txt" }
  Response 200: { "success": true, "path": "D:/transcriptions/lecture.txt" }
```

## 9. 潜在挑战与应对

| 挑战 | 应对方案 |
|------|----------|
| 进度字典仅保留最后 200 词 | 前端客户端侧累积。每个轮询返回新的 words，前端自增追加。漏掉的轮询由下一次补全。 |
| 用户刷新页面丢失轮询状态 | 加载时检查 `/transcribe_list` 恢复活跃任务 |
| 标签页后台时浏览器限制定时器 | `setTimeout` 链代替 `setInterval`。后台时降频到 10 秒。 |
| localStorage 配额（5MB） | 仅存最近 10 条完整结果 + 预览片段。超过 80% 时警告。 |
| AI agent 的阻塞 `/transcribe` 与前端队列共存 | 保留阻塞端点，不经过队列直接处理。使用时 GPU 可能正被前端任务占用。 |
| 超大文件上传（3 小时视频 ~GB 级） | 前台上传时显示进度百分比（fetch 不支持原生上传进度，可用 XMLHttpRequest 或分块）。 |
| 并发转写冲突 | 排队机制确保一次一个。后端加锁防并发。 |
